## ? Graph Algorithms & Union-Find
1. **Number of Islands II (LC #305)**: Dynamically adding nodes and tracking connected components in a grid, solved with **Union-Find (DSU)**, path compression, and union by rank[reference:0].
2. **Disjoint Set Union (DSU) Problem**: Asked in an elimination round; required an optimal solution with thorough follow-ups[reference:1].
3. **Parallel Courses (LC #1136) / Course Schedule II (LC #210)**: Topological sort and BFS/DFS problems related to course dependencies[reference:2].
4. **Multiple-Source BFS**: A problem where a BFS was required to propagate from multiple sources simultaneously[reference:3].
5. **Bus Routes (LC #815)**: Finding the minimum number of buses required to reach a destination by modeling bus stops and routes as a graph and performing BFS[reference:4].
6. **Alien Dictionary (LC #269)**: Deriving a character order from a list of words using a graph and topological sort (Kahn's algorithm)[reference:5].
7. **Clone Graph (LC #133)**: Deep copy of an undirected graph using BFS/DFS[reference:6].
8. **Minimum Edge Reversals**: A problem where you need to reverse the minimum number of edges so that every node is reachable from a given source[reference:7].
9. **Grid Robots & Nearest Blockers**: Precomputing nearest blockers in four cardinal directions for a 2D grid to answer movement queries in O(1)[reference:8].

13. **Container With Most Water (LC #11)**: Two-pointer technique on array heights to maximize area[reference:12].
14. **Product of Array Except Self (LC #238)**: Prefix and suffix products without division[reference:13].

16. **Valid Parentheses (LC #20)**: Stack-based validation[reference:15].

18. **Word Search (LC #79)**: Modified version with 8-direction movement and path constraints[reference:17].

## ? Trees & Binary Search Trees (BST)
29. **Tree DP (Root vs Leaf Race)**: Given a tree with parent pointers, two parties race from root and leaf; compute maximum points based on arrival times using DFS distance calculation[reference:28].
30. **Insert into Complete Binary Tree (LC #919)**: Using BFS level-order traversal to find the insertion point[reference:29].

34. **Lowest Common Ancestor (LCA)**: Tree-based LCA problems frequently appear.

## ? Preparation Tips Based on Recent Experiences
- **DSA rounds are elimination-focused** and expect optimized solutions, not just brute force[reference:62].
- **Graphs and Union-Find** are heavily emphasized; be comfortable with DSU, BFS/DFS, and topological sort[reference:63].
- **DP appears at medium-hard levels**; practice state definition and space optimization[reference:64][reference:65].
- **For L5A, system design rounds are critical** and often involve real-world Uber scenarios like ride-matching, surge pricing, and real-time metrics[reference:66].
- **LLD rounds require complete working code** with good design patterns and test cases[reference:67].
- **Use LeetCode's Uber-tagged questions** as a primary resource—many candidates report encountering exact or similar problems[reference:68].
